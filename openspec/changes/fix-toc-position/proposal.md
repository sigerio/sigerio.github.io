# 提案：固定文章目录位置并优化滚动条

## 变更标识
- **change-id**: `fix-toc-position`
- **创建日期**: 2025-11-27
- **状态**: 草案

## 背景

当前文章目录使用 `position: sticky` 定位，会在一定范围内随页面滚动。用户希望目录完全固定在视口右侧，不随页面滚动，同时优化滚动条样式使其更加明显。

## 问题摘要

### 问题1：目录位置随页面滚动 [高]

**现状**：使用 `position: sticky`，目录在容器范围内会随页面滚动

**期望**：使用 `position: fixed`，目录完全固定在视口右侧

### 问题2：滚动条不够明显 [中]

**现状**：使用系统默认滚动条样式，可能不够明显

**期望**：优化滚动条样式，使其更加可见

## 范围

| 范围 | 包含 |
|------|------|
| 定位调整 | TOC 改为 fixed 固定定位 |
| 样式优化 | 滚动条样式美化 |

## 成功标准

1. 文章目录固定在视口右侧，不随页面滚动
2. 目录过长时显示明显的滚动条
3. 移动端响应式适配正常
